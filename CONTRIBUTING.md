🤝 Contributing to Todo BLoC Guide
Thank you for your interest in contributing! This project is meant to be a learning resource, so contributions that improve clarity and educational value are especially welcome.

How to Contribute
Fork the repository
Create a feature branch (git checkout -b feature/amazing-feature)
Commit your changes (git commit -m 'Add amazing feature')
Push to the branch (git push origin feature/amazing-feature)
Open a Pull Request
Code Style
Follow the existing comment style — explain WHY, not just WHAT
Use descriptive variable names
Keep cubits focused on single responsibility
Always use Equatable for state classes
Write tests for new cubit methods
Commit Messages
Use present tense: "Add feature" not "Added feature"
Be descriptive: "Add search filter to todo cubit" not "update code"
